### 学习内容

- `Main.java` 使用 `import cn.abigmelon.entity.*;`，说明今天可能在练习同一包下多个类的批量导入。相比只导入 `Person`，通配导入能让 `Main` 同时直接使用 `Person` 和 `Human`。
- `Main.java` 新增 `Person p5 = new Person(); p5.hello(); p5.x();`，意图是围绕“访问权限控制”观察哪些成员能被外部包中的类调用。不过当前 `Person.java` 中没有 `x()` 方法，代码目前可能无法编译，这一处很适合用来反查访问修饰符和方法定义是否一致。
- `Human.java` 中 `private String name; private int age; private String gender;` 与注释“为了防止对象被外部乱改，需要用到类的封装”直接对应，说明今天学习重点转向封装：字段不再直接暴露给外部，而是通过方法控制访问。
- `Human` 提供 `setName`、`setAge`、`setGender`，说明正在练习 setter 方法。代码中 `this.name = name;`、`this.age = age;`、`this.gender = gender;` 展示了使用 `this` 区分成员变量和形参。
- `Human` 提供 `getName`、`getAge`、`getGender`，说明正在练习 getter 方法，用公开方法读取私有字段。
- `Human.newInstance()` 是一个 `public static` 方法，内部 `new Human()` 后返回对象，体现了静态工厂方法的基本形式。注释中提到“甚至可以把构造方法也改成私有的，通过内部方法来构造对象”，说明可能已经开始接触“控制对象创建入口”的思想。
- `private static Person Instance;` 与 `getInstance()` 中的空值判断，展示了可能的单例模式雏形：用静态字段保存一个对象，只在第一次调用时创建。不过这里字段类型和方法返回类型是 `Person`，而不是当前类 `Human`，更像是练习时误用了类型。

### 关键代码理解

- `private` 字段只能在类内部直接访问。`Human` 把 `name`、`age`、`gender` 设为私有后，外部代码不能再写 `h1.age = 18`，而应通过 `h1.setAge(18)` 这样的公开方法修改状态。
- setter 的核心作用不是简单替代直接赋值，而是给后续校验留入口。例如 `setAge(int age)` 现在直接赋值，将来可以加入 `age >= 0` 之类的判断。
- getter 的作用是提供可控读取。即使字段是 `private`，外部仍然可以通过 `getAge()` 获取值，但类可以决定是否返回原值、计算值或限制暴露。
- `public static Human newInstance()` 不依赖已有对象，可以通过 `Human.newInstance()` 调用。它把“创建对象”封装进类内部，是理解静态方法和对象构造关系的一个好例子。
- `getInstance()` 中 `if (Instance == null) { Instance = new Person(); } return Instance;` 表示第一次调用时创建对象，后续调用复用同一个静态对象。这是单例思想的核心结构，但当前代码返回的是 `Person`，与 `Human` 类的主题不一致。

### 易错点与待修正

- `Main.java` 调用了 `p5.x()`，但当前 `Person.java` 没有定义 `x()` 方法。若直接编译，会出现“找不到符号”一类错误；需要补充对应方法，或删除这行调用。
- `Human.getInstance()` 使用 `private static Person Instance` 并返回 `Person`，而它位于 `Human` 类中。若目标是练习 `Human` 的单例，应改为 `private static Human instance;` 和 `public static Human getInstance()`；若目标是演示跨类持有对象，则建议改名避免误解。
- `Instance` 变量名首字母大写，不符合 Java 普通常量以外变量的小驼峰命名习惯。静态普通字段建议写作 `instance`。
- `setAge(int age)` 目前没有校验，封装的优势还没有体现出来。可以加入年龄不能为负数的判断，进一步理解 setter 的价值。
- `Person.info` 的注释被截断为“变成了类的属”，这不是语义错误，但会影响后续复盘阅读，建议之后补全为“变成了类的属性”。

### 明天建议

- 先修正 `p5.x()` 对应的方法缺失问题，明确今天想练的是 `public`、`private`、默认访问权限还是 `protected`。
- 把 `Human.getInstance()` 的类型统一成 `Human`，再观察 `new Human()`、`Human.newInstance()`、`Human.getInstance()` 三种创建方式的差异。
- 在 `setAge` 中加入简单校验，例如拒绝负数，体会封装不仅是“字段私有化”，还包括对状态变化进行约束。
- 可以新增一个小练习类，专门比较 `Person` 的 `public` 字段直接访问和 `Human` 的 `private` 字段加 getter/setter 两种写法。

## 本周待巩固

- 包导入方式：单类导入和通配导入的适用场景。
- 访问控制：`public`、`private`、默认访问权限在跨包调用时的差异。
- 封装：私有字段、getter/setter、状态校验之间的关系。
- 静态方法：通过类名调用，不依赖具体对象。
- 静态工厂方法与构造方法：对象创建入口可以被封装到类内部。
- 单例模式雏形：静态字段缓存对象、空值判断、返回同一个实例。
- 编译前一致性检查：调用的方法必须在对应类中真实存在。
