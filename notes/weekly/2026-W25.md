### 学习内容

- `Main.java` 使用 `package cn.abigmelon;`，`Person.java` 使用 `package cn.abigmelon.entity;`，并在主程序中通过 `import cn.abigmelon.entity.Person;` 引入实体类。代码能看出正在练习包声明、目录层级和类引用之间的一致关系。
- `public static void main(String[] args)` 作为程序入口，内部大量使用 `System.out.println` 输出变量、表达式和对象字段，说明当前练习以运行观察结果为主。
- 基本类型部分覆盖了 `int`、`long`、`double`、`float`、`char`、`boolean`，并练习了 `long` 后缀 `L`、`float` 后缀 `f`、数字下划线写法 `1_000_000`、十六进制 `0xA` 和八进制 `012`。
- 字符与编码部分通过 `char h = 65;`、`int j = '瓜';` 观察字符和整数编码之间的关系，能看出在理解 `char` 本质上可以和编码值互相对应。
- 字符串部分通过 `"一只" + "大西瓜" + true + 1.5` 练习字符串拼接，以及其他类型参与 `+` 运算时会转成字符串参与拼接。
- 控制流部分使用了新版 `switch` 表达式：既有 `case 100 -> 'A'` 这种直接返回，也有代码块分支中使用 `yield 'D'` 返回结果，说明正在学习 `switch` 表达式与传统 `switch` 语句的差别。
- 面向对象部分通过 `new Person()`、`Person p1 = new Person()`、`Person p3 = p1` 以及 `p1 == p2`、`p1 == p3` 的输出比较，练习对象创建、引用变量和引用相等性的区别。
- `Person` 类中定义了 `name`、`age`、`gender` 字段，以及 `hello()`、`sum()`、`setName()`、`getGender()` 等方法，说明已经进入类的字段、实例方法和对象状态修改的学习。
- `sum(int, int)` 与 `sum(double, double)` 展示了方法重载；多个 `Person` 构造方法展示了构造方法重载。
- `setName(String name)` 中使用 `this.name = name;`，`Person(int age, String gender)` 中使用 `this(null, age, gender);`，说明正在练习 `this` 区分成员字段与形参，以及构造方法之间的复用调用。
- `public static String info` 和 `public static void setInfo(String info)` 展示了静态成员属于类本身，`Main.java` 中通过 `Person.info = "hello";` 后再从对象引用读取，体现了所有对象共享同一个静态字段。

### 关键代码理解

- `Person p3 = p1;` 不会创建新对象，而是让 `p3` 指向 `p1` 指向的同一个对象。因此 `p1 == p3` 为 `true`；`p1 == p2` 比较的是两个不同对象引用，所以为 `false`。
- `final int b = 10;` 表示变量初始化后不能再重新赋值，适合用来理解常量和普通变量的差异。
- `var l = 10;` 是局部变量类型推断，编译器会根据右侧字面量推断出 `int`，但变量本身仍然有确定类型。
- `char grade = switch (score) { ... };` 表示整个 `switch` 产生一个值并赋给变量；当分支里有多条语句时，需要通过 `yield` 指定该分支最终返回的值。
- `this(null, age, gender);` 必须出现在构造方法的第一条语句位置，用于复用同类中的其他构造方法，减少重复初始化逻辑。
- `Person.info` 是静态字段，推荐用类名访问；虽然代码中 `p2.info` 能访问，但容易让人误以为它属于单个对象。

### 易错点与待修正

- `System.out.println(p4.age);` 旁边注释写着“默认为0”，但 `Person` 类中 `age` 已初始化为 `18`，所以当前输出应为 `18`。如果想观察成员变量默认值，需要去掉字段初始化，或新增一个未初始化的 `int` 字段。
- `Person.info` 这种静态成员建议统一通过类名访问；通过对象引用访问静态字段虽然能编译，但不利于区分类变量和实例变量。
- `public` 字段可以直接被外部修改，例如 `p4.name = "melon";`。后续学习封装时，可以把字段改为 `private`，再通过 getter/setter 控制访问。
- `Main.java` 当前把很多知识点集中写在一个 `main` 方法里。随着内容增加，可能需要按主题拆分练习类，避免后续复盘和调试混在一起。
- `README.md` 在当前终端读取时出现中文乱码显示，但 Git 中的 Java 文件内容可以正常显示中文；后续如果继续维护 README，建议确认编辑器和文件编码统一使用 UTF-8。
