## 本周概览

本周目前主要推进到 Java 面向对象的中段内容：在已有类与对象、字段、方法、构造方法、`static`、封装和方法重载的基础上，继续补充了访问控制、继承、`final` 类、抽象类、接口、枚举、`record`、`Object` 常用方法重写、`Cloneable` 与浅克隆、`instanceof` 模式匹配等内容。代码重点从“创建对象并调用方法”扩展到了“类之间如何形成继承/实现关系，以及对象如何比较、打印、克隆和表达状态”。

### 学习内容

- 继承与访问控制：`student extends Person`、`teacher extends Person` 表明今天开始练习父类/子类关系；`Person.age` 改为 `protected`，说明开始区分 `public` 暴露字段与“允许子类访问但不完全公开”的成员可见性。
- 构造方法链：`teacher()` 中调用 `super(18)`，对应父类存在有参构造时，子类构造方法需要显式或隐式完成父类初始化。
- `final` 类：`teacher` 被声明为 `final class`，表示该类不能再被继承，用于练习继承边界。
- 抽象类：`Abstract` 中定义 `abstract void exam()`，同时保留普通方法 `test()`，说明抽象类既可以规定子类必须实现的能力，也可以提供可复用的具体行为。
- 接口：`Study1`、`Study2` 定义能力契约，`student implements Study1, Study2` 体现一个类可以实现多个接口；`Study3 extends Study2` 体现接口之间也可以继承。
- 接口默认方法和静态方法：`Study1.test001()` 使用 `default`，并调用 `static inner()`，说明开始接触接口中除抽象方法外的默认实现和工具方法。
- `Object` 方法：`Person` 重写 `equals` 和 `toString`，`Main` 中比较 `p01 == p02` 与 `p02.equals(p01)`，体现“引用是否同一个对象”和“对象内容是否相等”的区别。
- 克隆：`Person implements Cloneable` 并重写 `clone()` 调用 `super.clone()`，`Main.testClone()` 比较克隆前后 `==` 与 `equals`，说明今天学习了对象浅拷贝的基本调用路径。
- `instanceof` 模式匹配：`if (person instanceof student student)` 在判断成功后直接得到转换后的变量，减少了传统强制类型转换步骤。
- 枚举：`Status` 枚举常量带参数，内部用 `private final String name` 保存展示值，并提供 `getName()`，说明枚举不仅是固定常量列表，也可以拥有字段、构造方法和普通方法。
- `record`：`TestData` 用一行声明数据结构，`Main` 中调用 `order1.id()` 并打印对象，说明开始了解 `record` 自动生成访问器、`equals`、`hashCode`、`toString` 的特点。

### 关键代码理解

- `Person p01 = new Person("melon", 18, "male"); Person p02 = new Person("melon", 18, "male");` 后，`p01 == p02` 为引用比较，通常为 `false`；重写 `equals` 后，字段内容相同则可以返回 `true`。
- `clone()` 能够创建另一个对象实例，因此 `p001 == clone` 应为 `false`；如果字段值相同且 `equals` 按字段比较，则 `p001.equals(clone)` 可以为 `true`。当前示例是浅克隆，若以后字段中包含可变引用对象，需要继续区分浅拷贝和深拷贝。
- `student` 同时 `extends Person` 和 `implements Study1, Study2`，体现 Java 单继承、多接口实现的规则。
- `Status.RUNNING("...")` 实际会调用枚举的私有构造方法，枚举构造方法不能在外部直接 new。
- `record TestData(int id, String product, String address)` 会自动生成 `id()`、`product()`、`address()` 访问器，而不是传统 JavaBean 风格的 `getId()`。

### 易错点与待修正

- `Person.toString()` 当前返回字符串缺少右侧 `]`，打印结果可能不完整；后续可以修正为 `Person[name=..., age=..., gender=...]`。
- `Person.equals()` 中直接调用 `this.name.equals(person.name)` 和 `this.gender.equals(person.gender)`，如果字段为 `null` 可能触发 `NullPointerException`；后续学习 `Objects.equals` 后可改进。
- `Person` 声明 `Cloneable` 并调用 `super.clone()` 是浅克隆示例，后续遇到引用类型字段时不能误以为所有内部对象都会被复制。
- `student.java`、`teacher.java` 类名使用小写开头，不符合 Java 类名通常使用大驼峰的约定；建议后续改为 `Student`、`Teacher`，并同步文件名。
- `Main.java` 中存在未使用的 `import java.util.TreeSet;`，可能是为后续集合学习预留；如果暂时不用，可以删除以保持示例干净。
- 源码注释在终端显示有编码乱码迹象，可能是文件编码与查看工具编码不一致；不影响本次结构复盘，但后续建议统一 IDE 和文件编码为 UTF-8。
